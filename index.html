Require JS

Define

define("my/module", function () {

  return {
    prop: "something",
    doSomething: function () {}
  }
});

Require JS

Consume

require(["my/module"], function (myModule) {
    myModule.doSomething();
});

Benefits?

  • Code is in modules
  • editing on the fly

Problems?

  • Lot of config
  • A lot of boilerplate
  • Can be slow during devs
  • Many libs don't support it
  • Still requires build for production
  • A lot of config!

CommonJS

module.exports = {
  prop: "something",
  doSomething: function () {}
}

CommonJS

var myModule = require('myModule');
myModule.doSomething();

CommonJS

Benefits?

  • Simplier
  • Less config
  • NPM and (almost) everything in it

CommonJS

Problems?

  • Requires a compile step
  • Not every NPM module works
  • Sadly, not every lib is in NPM

ES6

export default {
  prop: "something",
  doSomething: function () {}
}

ES6

import myModule from "myModule";
myModule.doSomething();

ES6

Benefits?

  • Simplier
  • Less config
  • Future proof

ES6

Problems?

  • Requires a compile step
  • Still new
  • Years away from working in the browser
